## 第一章 平台是什么

### 1.1 一句话定义

**工厂工作流平台** = 用可视化流程图编排业务自动化 + AI 能力，并对外提供对话、API、消息推送等入口。

可理解为：**n8n（流程编排）+ Dify（AI 应用）** 的组合，面向工厂/企业内部场景。

### 1.2 核心概念

| 概念 | 说明 |
|------|------|
| 工作流 | 节点 + 连线的自动化流程 |
| 节点 | 做一件具体的事（触发、AI、发消息等） |
| 触发器 | 流程起点（手动、定时、对话、Webhook 等） |
| 知识库 | 上传文档，供 AI 检索引用（RAG） |
| 对话应用 | 把工作流发布成聊天机器人 / API |
| 执行中心 | 查看每次运行的详细日志 |
| 渠道接入 | 企业微信、飞书等消息通道 |

### 1.3 菜单地图

- **控制台** — 总览统计  
- **工作流** — 创建、列表、编辑器  
- **执行中心** — 运行记录与节点输出  
- **会话中心** — 对话历史、人工回复  
- **渠道接入** — 企业微信 / 飞书  
- **连接器** — HTTP、数据库等  
- **AI · 知识库** — 文档导入与检索  
- **AI · 模型** — 大模型 / Embedding 配置  
- **AI · 应用** — 对话应用与 API Key  
- **系统管理** — 租户与用户

## 第二章 30 分钟入门

### 目标

完成：**手动触发 → AI 对话 → 执行中心看到结果**

### 步骤

1. **工作流** → 点 **新建**（名称可留空）  
2. 编辑器左侧 **模板** → 选 **「AI 智能问答」**  
3. 顶栏：**保存** → **发布**  
4. 顶栏：**运行** → **执行中心** 查看最新记录  

### 要点

- 必须 **发布** 后，外部触发（定时/Webhook/对话）才生效  
- 手动 **运行** 注入测试数据：`content: "手动测试"`  
- AI 失败：检查 **AI · 模型** 是否配置  

### 练习

- [ ] 改 AI「系统提示」后再运行，对比输出  
- [ ] 在执行中心找到 AI 节点的 `text` 字段

## 第三章 工作流编辑器

### 界面

```
节点库（左） | 画布（中） | 配置面板（右）
顶栏：保存 | 发布 | 运行 | 自动排列
```

### 基本操作

| 操作 | 方法 |
|------|------|
| 加节点 | 左侧拖拽或点击 |
| 连线 | 右侧圆点拖到下一节点 |
| 配参数 | 点节点 → 右侧面板 |
| 插变量 | 变量选择器点击插入 |
| IF 分支 | 绿=是，红=否 |

### 变量语法

| 写法 | 含义 |
|------|------|
| `{{trigger.content}}` | 触发带来的主要内容 |
| `{{steps.__last__.text}}` | 上一步 AI 文本输出 |
| `{{steps.__last__.chunks}}` | 上一步知识库检索结果 |
| `{{steps.节点ID.字段}}` | 指定节点输出 |
| `{{trigger.firedAt}}` | 定时触发时间 |

### 节点速查

**触发器：** 手动、对话、Webhook、定时、企微、飞书  

**逻辑：** IF、Switch、设置变量、延迟、并行、汇合、子工作流  

**AI：** AI 对话、知识库检索、AI Agent  

**集成：** HTTP 请求、消息推送

## 第四章 知识库（RAG）

### 流程

1. 左侧 **创建知识库** → **选中**（高亮）  
2. **上传** .txt / .md / .csv 或 **粘贴导入**  
3. **检索评测**：先用 **关键词**（向量需 Embedding）  
4. 工作流：**知识库检索 → AI 对话**  

### RAG 标准结构

```
触发 → 知识库检索 → AI 对话（引用 {{steps.__last__.chunks}}）
```

### 检索模式

| 模式 | 前提 |
|------|------|
| 关键词 | 无需 Embedding，立即可用 |
| 向量 | 需 Embedding 模型 + 重向量化 |

## 第五章 对话应用

### 是什么

把工作流包装成 **聊天产品**（网页 + API + 会话记录）。

### 发布步骤

1. 工作流含 **对话触发** + AI（推荐：对话 → 知识库 → AI）  
2. 工作流 **保存 → 发布**  
3. **AI · 应用** → 新建 → 选工作流 → **发布**  
4. 复制：聊天链接 / API Key / 嵌入代码

## 第六章 AI 模型配置

### 两个接口

| 用途 | 字段 | 接口 |
|------|------|------|
| 对话 | 默认 Chat 模型 | `/chat/completions` |
| 向量 | 默认 Embedding 模型 | `/embeddings` |

Chat 能通 ≠ 向量能用。自建网关若无 `/embeddings`，知识库请用 **关键词检索**。

### 配置路径

**AI · 模型** → 填 Base URL、API Key、Chat 模型、Embedding 模型 → 设为默认 → 保存  

向量化：知识库文档点 **重向量化**，左侧显示「X 已向量化」。

模板：**Webhook 告警**。

## 第九章 运维与排错

| 现象 | 处理 |
|------|------|
| 向量检索无结果 | 用关键词；配 Embedding 后重向量化 |
| 对话无回复 | 看执行中心；确认工作流已发布 |
| 定时没触发 | 工作流必须发布 |
| 飞书发不出 | 渠道配置 + chat_id |
| AI 报错 | 检查 AI 模型 Key/地址 |

**发布 Checklist：** 保存 → 发布工作流 → 模型可用 → 知识库已导入 → 应用已发布
